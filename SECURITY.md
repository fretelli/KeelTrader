# Security Policy

<a id="en"></a>
[English](#en) | [中文](#zh-cn)

## Supported Versions

Only the latest `main` branch is supported.

## Reporting a Vulnerability

Please do **not** open a public GitHub Issue for security reports.

Use one of the following:

1. GitHub Security Advisories (preferred): `Security` tab → `Report a vulnerability`
2. (Optional) Email a private contact address if you operate one (replace this line with a real address)

Include:

- Affected component (Web/API)
- Steps to reproduce / PoC (if available)
- Impact assessment
- Suggested fix (if you have one)

We will acknowledge receipt and work with you on a coordinated disclosure timeline.

## 支持范围

## 报告漏洞

请 **不要** 通过公开的 GitHub Issue 报告安全问题。

请使用以下方式之一：

1. GitHub Security Advisories（推荐）：`Security` 选项卡 → `Report a vulnerability`

请尽量包含：

- 受影响组件（Web/API）
- 复现步骤 / PoC（如有）
- 影响评估
- 修复建议（如有）

我们会确认收到并与你协作制定负责任披露时间线。
